# Deployment Guide

This guide explains how to deploy the **AI-Enterprise Multi-Agent Knowledge Assistant** application in production.

---

## 1. Backend Deployment (Render)

We deploy the FastAPI backend service using **Render**. Render supports Blueprint specification via the `render.yaml` file located in the root of this project.

### Step-by-Step Deployment:
1. Log in to your [Render Dashboard](https://dashboard.render.com).
2. Click **New** (top right) and select **Blueprint**.
3. Connect your GitHub repository: `https://github.com/HarshithanagashreeR/AI-Workspace-Enterprise-Multi-Agent-Knowledge-Assistant.git`.
4. Render will automatically detect the `render.yaml` configuration and list the services to create:
   - **Service Name**: `enterprise-backend` (Docker-based Web Service)
   - **Database Name**: `enterprise-db` (Free PostgreSQL instance)
5. During the prompt, set the following environment variables:
   - `OPENAI_API_KEY`: Set your valid OpenAI API key.
   - `FRONTEND_URL`: Set this to your deployed Vercel frontend URL: `https://frontend-beige.vercel.app` (or any custom domain you config).
6. Click **Apply** to start provisioning.
7. Once provisioned, Render will allocate a public URL for your backend (e.g. `https://enterprise-backend.onrender.com`).

---

## 2. Frontend Deployment (Vercel)

The React SPA is deployed on **Vercel** and built using Vite.

### Step-by-Step Deployment:
1. Navigate to the [Vercel Dashboard](https://vercel.com).
2. Click **Add New** -> **Project**.
3. Import your GitHub repository.
4. Set the project configuration:
   - **Framework Preset**: `Vite`
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build` (runs `tsc -b && vite build`)
   - **Output Directory**: `dist`
5. Configure the **Environment Variables**:
   - Add `VITE_API_URL` and set its value to your deployed Render API path (e.g., `https://enterprise-backend.onrender.com/api`).
6. Click **Deploy**.
7. Vercel will build and assign your production domain: `https://frontend-beige.vercel.app`.

## 3. Environment Variables Configuration Reference

### Backend Settings (Render / `.env`)
- `DATABASE_URL`: Automatically configured and linked to the Render PostgreSQL instance.
- `JWT_SECRET_KEY` & `JWT_REFRESH_SECRET_KEY`: Automatically generated by Render during blueprint setup.
- `OPENAI_API_KEY`: Required for multi-agent execution (if MOCK_LLM is false).
- `MOCK_LLM`: Set to `true` to run the entire backend RAG flow using high-fidelity mock LLM data completely for free. Defaults to `false`.
- `RATE_LIMIT_PER_MINUTE`: Defaults to `100`.
- `FRONTEND_URL`: CORS origin allowed to communicate with the backend. Must match the Vercel URL.
- `ENVIRONMENT`: Set to `production`.

### Frontend Settings (Vercel / `.env`)
- `VITE_API_URL`: Points to the backend endpoint + `/api` prefix (e.g. `https://enterprise-backend.onrender.com/api`).

## 4. How to Redeploy

### Automatic CD (Continuous Deployment)
- **Vercel & Render**: Both platforms are linked directly to your GitHub repository. Every push or merge to the `main` branch will automatically trigger a new production build and zero-downtime deployment.

### Manual Redeploy
- **Vercel**: Go to your project page on Vercel -> select **Deployments** -> click the three dots on the latest deployment -> select **Redeploy**.
- **Render**: Go to the web service page on Render -> click **Manual Deploy** -> select **Clear Cache & Deploy** or **Deploy Latest Commit**.
